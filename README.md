# 🇬🇧 English Learning Bot

Telegram-бот для изучения английского языка (aiogram 3.x). Тесты со словами,
глаголами и предложениями, аудирование по уровням CEFR, интервальное повторение (SRS),
адаптивная сложность и озвучка через TTS.

Построен на **чистой слоистой архитектуре**: логика вынесена из хендлеров, данные —
за репозиториями, внешние движки — за провайдерами.

## ✨ Возможности

Главное меню (нижняя панель): **📝 Тесты · 🎧 Аудирование · 🗣 Говорение · ⚙️ Настройки**

### 📝 Тесты (в чате, inline)
Одна кнопка → выбор: **Слова · Глаголы · Предложения · 🔁 Повторение**. Единый движок теста:
- каждый вопрос — **отдельным сообщением** (отвеченные помечаются ✅/❌ и остаются в чате — можно разобрать ошибки);
- **авто-переход**, **таймер-дедлайн** с авто-провалом (или без таймера);
- кнопка **⏹ Остановить тест** в нижней панели;
- **📖 Слова** — `school [skuːl] [скул]` → выбрать перевод (999 слов, A1–B2);
- **🔄 Глаголы** — выбрать формы V2/V3; после ответа варианты с переводом (`took / taken — брать`);
- **✍️ Предложения** — заполни пропуск `My ____ is …`; после ответа варианты с переводом (`song — песня`) и перевод всего предложения;
- **🔁 Повторение (N)** — слова, у которых подошёл срок по SRS (см. ниже).

### 🎧 Аудирование
- **Уровни A1–C1**, по **30 текстов** на уровень (150 всего; длина растёт по сложности);
- выбор уровня и текста — inline; управление — в нижней панели;
- **3 скорости** озвучки (🐢🚶🏃), листание текстов ⬅️➡️, **перевод**;
- голос диктора берётся из настроек.

### ⚙️ Настройки (сохраняются по пользователю)
- 🎚 **Сложность** — A1/A2/B1/B2/Все (фильтрует слова в тестах);
- ⏱ **Время на ответ** — 10/15/25 сек / без таймера;
- 🔢 **Вопросов в тесте** — 5/10/15;
- 🔊 **Голос озвучки** — 🇺🇸/🇬🇧 × муж/жен.

### 🧠 Обучающие механики
- **SRS (интервальное повторение)** — коробки Лейтнера: верный ответ отодвигает слово
  дальше (сегодня → +1 → +3 → +7 → +16 дней), ошибка возвращает в начало. Любой ответ
  по слову (обычный тест или повторение) кормит SRS.
- **Адаптивная сложность** — бот следит за точностью на уровне и после теста слов
  предлагает **⬆️ повысить** (≥85%) или **⬇️ понизить** (≤45%).

### 🗣 Говорение
Заглушка («скоро») — интерфейс STT-провайдера уже заложен.

## 🏗 Архитектура

Слоистая + элементы Clean Architecture. Поток зависимостей строго вниз:

```
Handlers (Telegram I/O)         handlers/*.py
      ↓
Services / Use Cases            services/*.py
      ↓
Repositories (абстракции)       repositories/*.py
      ↓
Data Sources (.py / JSON)       data/*.py, *.json
```
Плюс поперечные слои: **models** (доменные dataclass), **providers** (TTS/STT/AI за
интерфейсами), **keyboards/presenters** (представление), **callbacks** (единый формат
callback_data), **containers** (композиционный корень).

**Принципы:**
- Хендлер не знает, откуда данные; сервис не знает про Telegram; репозиторий не знает про бизнес-логику.
- Данные — через доменные модели, а не голые словари (в т.ч. состояние теста `QuizSession`).
- Источник данных абстрагирован: переход `.py → Google Sheets` = один новый класс-репозиторий, без правок в сервисах/хендлерах.
- Внешние движки (озвучка/распознавание/LLM) — за провайдер-интерфейсами.

### Структура проекта

```
bot.py                # точка входа: сборка контейнера, запуск polling
config.py             # настройки из .env (stdlib, без секретов в коде)
containers.py         # композиционный корень — единственное место с конкретными классами
callbacks.py          # единый формат callback_data (build/parse)
settings_catalog.py   # справочник опций настроек (значение + подпись)

models/               # доменные модели (dataclass)
  level, word, verb, text, sentence, settings, quiz (QuizSession+логика), listen
data/                 # контент как .py  (→ позже Google Sheets)
  words_data (999) · verbs_data (56) · sentences_data (500)
  sentence_words_data (417 переводов) · texts_data (150)
repositories/         # DAO
  base.py             # абстракции (async): Word/Verb/Text/Sentence + Settings/Stats/Srs
  pyfile/             # реализации из .py (контент, read-only)
  json_*_repo.py      # settings / stats / srs (read-write, JSON)
providers/            # tts(edge) · stt(stub) · ai(stub) — за абстракциями
services/             # бизнес-логика
  quiz_service (единый движок) · vocabulary · verb · sentence · listening
  settings · stats (адаптив) · srs (повторение)
presenters/           # форматирование сообщений (View)
keyboards/            # клавиатуры
handlers/             # Telegram I/O (тонкие)
  start · quiz_flow · listening · settings · fallback · utils
```

### Что где хранится
- **Контент** (слова/глаголы/предложения/тексты) — `data/*.py`, read-only.
- **Данные пользователя** — JSON (все в `.gitignore`): `settings.json`, `stats.json`, `srs.json`.
- **Кэш озвучки** — `tts_cache/` (по хэшу голос+скорость+текст).

## 🔌 Точки расширения

- **Google Sheets** вместо `.py`: написать `GoogleSheetsWordRepository(AbstractWordRepository)` и подменить строку в `containers.py`.
- **STT (говорение)**: реализовать `AbstractSTTProvider` (Whisper / облачный STT).
- **AI/LLM**: реализовать `AbstractAIProvider` (генерация вопросов, объяснение ошибок).
- Состояние теста типизировано (`QuizSession`), логика в модели — легко покрыть юнит-тестами.
